<task type="auto">
  <name>Task 1: Improve mobile section stepper touch targets</name>
  <files>components/learn/LessonPageContentV2.tsx</files>
  <action>
Increase mobile stepper dot sizes for better touch targets:
1. Change current dot from `h-8 w-8` to `h-10 w-10` (40px visual, with tap area expansion)
2. Change non-current dots from `h-6 w-6` to `h-8 w-8` (32px visual)
3. Add `min-h-[48px] min-w-[48px]` touch area wrapper around each dot button to ensure 48px tap target
4. Keep the visual dot sizes smaller inside the touch target for aesthetics
5. Increase connector line width from `w-4` to `w-6` for better visual balance

Why: WCAG recommends 44px minimum touch targets; 48px is the iOS/Android standard for comfortable tapping.
  </action>
  <verify>Run `npm run type-check` to ensure no TypeScript errors</verify>
  <done>Mobile stepper dots have 48px touch targets while maintaining visual proportions</done>
</task>

<task type="auto">
  <name>Task 2: Standardize section content spacing</name>
  <files>components/learn/LessonPageContentV2.tsx</files>
  <action>
Standardize spacing in renderSectionContent for all four sections (dialogue, vocabulary, grammar, practice):
1. Use consistent `space-y-6` for main section containers (currently mixed space-y-6, space-y-2, space-y-3)
2. Use consistent `space-y-2` for section header + description groups
3. Ensure section headers use consistent `space-y-3` before content
4. Standardize the completion screen spacing to match other sections

The pattern should be:
- Section wrapper: space-y-6
- Header group (icon + title + description): space-y-2
- Content gap from header: mt-6 or space-y-6
- Inner content: component-specific (leave as-is)

Do NOT change the vocabulary grid or grammar section internal spacing — those will be handled in Plan 02.
  </action>
  <verify>Visual inspection in dev server - sections should have uniform vertical rhythm</verify>
  <done>All four sections use consistent space-y-6 outer spacing with space-y-2 header groups</done>
